API overview

Most data in DDI can be accessed programmatically using a RESTful API allowing for integration with other resources. The API implementation is based on the Spring Rest Framework.

Web browsable API

The DDI API is web browsable, which means that:

  • the data is served in either human- or computer-friendly formats depending on whether a URL is viewed in a browser or is retrieved programmatically

  • many resources are hyperlinked so that it's possible to navigate the API in the browser.

As a result, developers can familiarise themselves with the API and get a better sense of the DDI data.

Browse the API

Throttling

The maximum number of requests from the same IP address is limited to 20 requests per second. Currently there is no limit on the total number of requests from the same IP.

The limit can be lifted for registered users, so please get in touch if you require a higher request rate.

Responses containing multiple entries have the following fields:

  • count is the number of entries in the matching set
  • datasets is an array of datasets.
  • facets is an array of facets.

Pagination

Responses containing multuple datasets are paginated to prevent accidental downloads of large amounts of data and to speed up the API.

The page size is controlled by the size parameter. Its default value is 20 datasets per page, and the maximum number of datasets per page is 100.

Examples

  • http://localhost:9091/dataset/search?query=human&start=0&size=50&faceCount=20
  • http://localhost:9091/dataset/search?query=human&start=0&size=20&faceCount=20

Output formats

The following output format is: JSON

Filtering

The API supports several filtering operations that complement the main DDI search functionality.

Filtering by search term

There is 1 url parameter: query

Examples

  • http://localhost:9091/dataset/search?query=human
  • http://localhost:9091/dataset/search?query=cancer

Filtering by omics type

The omics type can be specified by adding term in the query url parameter with key: "omics_type" (possible values: Proteomics, Metabolomics, Proteomics).

Examples

  • http://localhost:9091/dataset/search?query=human AND omics_type:"Proteomics"
  • http://localhost:9091/dataset/search?query=human AND omics_type:"Metabolomics"

Filtering by databases

The database can be specified by adding term in the query url parameter with key: "repository" (possible values: MassIVE, Metabolights, PeptideAtlas, PRIDE).

Examples

  • http://localhost:9091/dataset/search?query=human AND repository:"MassIVE"
  • http://localhost:9091/dataset/search?query=human AND repository:"Metabolights"
  • http://localhost:9091/dataset/search?query=human AND repository:"PeptideAtlas"
  • http://localhost:9091/dataset/search?query=human AND repository:"PRIDE"

Filtering by Organism

The organism can be specified by adding term in the query url parameter with key: "TAXONOMY" (possible values must be the TAXONOMY id: 9606, 10090...).

Examples

  • http://localhost:9091/dataset/search?query=human AND TAXONOMY:"9606"
  • http://localhost:9091/dataset/search?query=human AND TAXONOMY:"10090"

Filtering by Tissue

The tissue can be specified by adding term in the query url parameter with key: "tissue" (possible values: Liver, Cell culture, Brain, Lung...).

Examples

  • http://localhost:9091/dataset/search?query=human AND tissue:"Brain"
  • http://localhost:9091/dataset/search?query=human AND tissue:"Cell culture"

    • Filtering by Disease

    The disease can be specified by adding term in the query url parameter with key: "disease" (possible values: Breast cancer, Lymphoma, Carcinoma, prostate adenocarcinoma...).

    Examples

    • http://localhost:9091/dataset/search?query=human AND tissue:"Breast cancer"
    • http://localhost:9091/dataset/search?query=human AND tissue:"Lymphoma"

      • Filtering by Modifications (in proteomics)

      The expert database can be specified by adding term in the query url parameter with key: "disease" (possible values: Breast cancer, Lymphoma, Carcinoma, prostate adenocarcinoma...).

      Examples

      • http://localhost:9091/dataset/search?query=human AND tissue:"Breast cancer"
      • http://localhost:9091/dataset/search?query=human AND tissue:"Lymphoma"

        • Filtering by external ids

        The external id is an id assigned to a sequence in one of the Expert Databases, which is imported into DDI as a cross-reference.

        The external id can be specified by setting the external_id url parameter.

        Examples

        • http://rnacentral.org/api/v1/rna/?external_id=Stap.epid._AF269831 (SRPDB)
        • http://rnacentral.org/api/v1/rna/?external_id=MIMAT0000091 (miRBase)
        • http://rnacentral.org/api/v1/rna/?external_id=OTTHUMG00000172092 (Vega)
        • http://rnacentral.org/api/v1/rna/?external_id=Lepto_inter_Lai566 (tmRNA Website)

        Combined filters

        Any filters can be combined to narrow down the query using the & symbol as a separator, which acts as the logical AND operator. More logical operators will be supported in the future.

        Examples

        • http://rnacentral.org/api/v1/rna/?database=srpdb&min_length=200
        • http://rnacentral.org/api/v1/rna/?min_length=1&max_length=4

        Genome annotations

        The API provides an endpoint for retrieving annotations based on genomic coordinates for a number of species.

        The response content is similar to that provided by the Ensembl REST API and can be directly used to visualise the data using the Genoverse HTML5 genome browser.

        The genome location should be in the chromosome:start-end format and may contain optional commas.

        Examples

        • http://rnacentral.org/api/v1/overlap/region/homo_sapiens/2:39,745,816-39,826,679

        Example script

        A common task is finding whether some sequence of interest has an DDI id.

        An example Python script for looking up DDI unique sequence ids (URS) is provided and can be used in more complicated programs.

        Instead of sending the entire sequence over the network to the DDI servers, the script only submits the md5 value calculated based on the sequence. The md5 value is 32 characters long and is unique for each sequence. Using md5 saves traffic and decreases the response time.

        Internally DDI uses the Digest::MD5 Perl module for computing the md5 values. Additional notes:

        • all sequences are uppercase
        • md5 is computed for DNA sequences (so all U's should be replaced with T's).

        An example Python script showing how to retrieve the DDI id for an RNA sequence.

        

        Usage:

        python retrieve_rnacentral_id.py

        """

         

        import hashlib

        import requests # pip install requests

         

         

        def get_md5(sequence):

        	"""

        	Calculate md5 for an RNA sequence

        	"""

        	# DDI stores DNA md5 hashes

        	sequence = sequence.replace('U','T')

        	# get the md5 digest

        	m = hashlib.md5()

        	m.update(sequence)

        	return m.hexdigest()

         

        # get the DDI id

        def get_rnacentral_id(md5):

        	"""

        	Parse json output and return the DDI id.

        	"""

        	url = 'http://rnacentral.org/api/v1/rna'

        	r = requests.get(url, params = {'md5': md5})

        	data = r.json()

        	if data['count'] > 0:

        		return data['results'][0]['rnacentral_id']

        	else:

        		return 'DDI id not found'

         

        # This sequence has an DDI id

        sequence = 'CUGAAUAAAUAAGGUAUCUUAUAUCUUUUAAUUAACAGUUAAACGCUUCCAUAAAGCUUUUAUCCA'

        md5 = get_md5(sequence)

        print get_rnacentral_id(md5)  # URS00002C9E9D

         

        # This sequence doesn't have an DDI id

        sequence = 'Not an RNA sequence'

        md5 = get_md5(sequence)

        print get_rnacentral_id(md5) # DDI id not found

        Cross domain requests

        To use the API in a javascript application, please use jsonp requests.

        Example using jQuery

        $.ajax({
    url: 'http://rnacentral.org/api/v1/rna/URS0000000001?format=jsonp',
    dataType: 'jsonp',
    jsonp: false,
    jsonpCallback: 'callback',
    success: function(data){
        // do something with the data
        console.log(data.rnacentral_id + ', ' + data.sequence);
    },
});

        The trailing slash in all urls used in the API is optional.
